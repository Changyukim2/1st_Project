기존 **안정성 중심 기업등급**을 보완하기 위해,  
**활동성(FAI)** 과 **수익성(FPI)** 을 결합한 중소기업 세분화 모델을 설계하고 검증한 프로젝트입니다.

이 프로젝트의 목표는 기존 등급체계만으로는 드러나지 않는  
**숨은 우량 중소기업**과 **잠재 우량고객**을 데이터 기반으로 발굴하는 것이었습니다.

---

## 프로젝트 개요

iM뱅크는 사업보고서 전반에서 **중소기업 지원 확대**와 **지역경제 활성화**를 주요 전략 방향으로 제시하고 있습니다.  
하지만 기존 기업등급체계는 안정성과 신뢰도 중심으로 설계되어 있어, 리스크 관리에는 강점이 있지만  
중소기업 내부의 실제 **영업활동성**, **수익창출력**, **성장 잠재력** 차이를 충분히 드러내기에는 한계가 있었습니다.

이에 따라 본 프로젝트에서는 기존 등급을 유지하되, 이를 보완하는 새로운 관점으로  
**활동성 지수(FAI)** 와 **수익성 지수(FPI)** 를 설계하고,  
기업을 보다 정교하게 세분화하는 모델을 제안했습니다.

---

## 문제 정의

기존 등급체계만으로는 다음과 같은 한계가 존재한다고 보았습니다.

- 동일한 기존 등급 내에서도 실제 영업활동성과 수익성 차이가 클 수 있음
- 자산 규모는 작지만 수익 효율이 높은 기업이 기존 등급에서 충분히 드러나지 않을 수 있음
- 중소기업 지원 전략을 정교화하려면 안정성 외의 추가 지표가 필요함

따라서 본 프로젝트는 다음 질문에서 출발했습니다.

> **기존 안정성 중심 등급체계를 유지하면서, 활동성과 수익성을 결합해 중소기업 집단 내부를 더 정밀하게 구분할 수 있는가?**

---

## 사용 데이터

- 분석 대상: 이상치를 제외한 **0원 ~ 5,000억 원 규모 기업 집단**
- 대상 기준: 법적 기준상 중소기업 범위에 해당하는 기업
- 처리 방식: 전처리 및 이상치 제거를 통해 비교 가능성 확보

---

## 핵심 아이디어

기업을 단순히 **안정적인가** 로만 평가하지 않고, 아래 두 축으로 재해석했습니다.

- **FAI (Firm Activity Index)**  
  기업의 외형적 활동 수준
- **FPI (Firm Profitability Index)**  
  기업의 은행 수익 기여 및 효율 수준

이 두 지수를 결합해 기업을 4개 세그먼트로 분류했습니다.

| Segment | 의미 |
|---|---|
| **Star** | 고활동 · 고수익 |
| **Cash Cow** | 저활동 · 고수익 |
| **Potential** | 고활동 · 저수익 |
| **Standard** | 저활동 · 저수익 |

---

## 지표 설계

### 1. 활동성 지수 (FAI)

활동성 지수는 세 가지 하위 요소를 동일 가중치로 반영했습니다.

#### 규모점수
- 총예금잔액
- 총대출잔액
- 총채널거래금액
- 총외환금액
- 총카드사용금액

#### 빈도점수
- 총채널거래건수
- 총외환거래건수

#### 다양성점수
- 예금, 대출, 카드, 외환 이용 여부를 기반으로 산출한 상품다양성개수

최종 식은 다음과 같습니다.

```text
FAI = (규모점수 + 빈도점수 + 다양성점수) / 3


2. 수익성 지수 (FPI)

수익성 지수는 두 가지 하위 요소를 반영했습니다.

수익기여점수
예대마진
추정 비이자수익
수익효율점수
자산대비수익률(%)

최종 식은 다음과 같습니다.

FPI = (수익기여점수 + 수익효율점수) / 2
3. 점수화 방식

금액·건수형 변수는 분포 왜곡이 크기 때문에 다음 절차를 적용했습니다.

log1p 변환
MinMaxScaler 정규화
각 변수별 score 생성
하위 점수 평균
최종 FAI / FPI 산출
세그먼트 분류 방식

FAI와 FPI의 중앙값을 기준으로 4사분면 분류를 수행했습니다.

FAI >= median and FPI >= median → Star
FAI < median and FPI >= median  → Cash Cow
FAI >= median and FPI < median  → Potential
FAI < median and FPI < median   → Standard

중앙값을 사용한 이유는 금융 데이터 특성상 0 값이 많고 분포 왜곡이 심해,
평균보다 중앙값이 더 안정적인 기준이었기 때문입니다.

검증 방법

데이터는 등분산성을 충족하지 않았고,
0 값과 왜도가 큰 금융 데이터 특성을 고려해 비모수 검정을 사용했습니다.

사용한 주요 검정
Kruskal-Wallis H 검정
Dunn 사후검정
Mann-Whitney U 검정
카이제곱 검정
주요 가설 검정
1. 기존 등급은 FAI와 FPI를 충분히 설명하는가?
기존 등급별 FAI, FPI 차이는 모두 유의함
하지만 기존 등급만으로는 고객 특성을 완전히 설명하기 어려움
2. 기존 등급과 신규 4사분면 분류는 같은 정보를 주는가?
두 체계는 유의한 연관성을 가짐
하지만 분포가 완전히 일치하지 않아 신규 세그먼트가 보완 정보 제공
3. 동일한 기존 등급 내부에서도 활동성 차이가 존재하는가?
모든 기존 등급 내에서 사분면별 FAI 차이가 유의함
4. 동일한 기존 등급 내부에서도 수익성 차이가 존재하는가?
모든 기존 등급 내에서 사분면별 FPI 차이가 유의함
5. 활동성 축은 실제로 고·저 활동 집단을 구분하는가?
고활동 집단과 저활동 집단 간 FAI 차이 유의
평균 FAI 약 1.8배 차이
6. 수익성 축은 실제로 고·저 수익 집단을 구분하는가?
고수익 집단과 저수익 집단 간 FPI 차이 유의
평균 FPI 차이 약 8.6%
7. 알짜(Cash Cow)는 스타(Star)보다 자산 대비 수익 효율이 높은가?
Mann-Whitney U 검정 결과 유의
자산 규모는 더 작지만 자산 대비 수익 효율은 더 높게 나타남
8 ~ 10. 알짜 그룹 내부에도 숨은 우량고객이 존재하는가?
알짜 그룹 내부에서도 기존 등급별 FPI 차이가 존재
특히 기존 일반 등급이면서 알짜인 고객군이 의미 있는 규모로 존재
보정된 활동 수익 밀도 기준으로도 고효율 고객 존재 가능성 확인
주요 분석 결과
1. 기존 등급은 유효하지만 충분하지는 않음

기존 등급체계는 안정성 판단 기준으로는 의미가 있었지만,
중소기업 내부의 실제 활동성과 수익 효율 차이를 세밀하게 설명하지는 못했습니다.

2. 신규 세그먼트는 기존 등급을 보완함

신규 4사분면 분류는 기존 등급과 관련은 있으나 동일하지 않았고,
같은 기존 등급 안에서도 기업 간 차이를 더 잘 드러냈습니다.

3. Cash Cow 세그먼트가 핵심 인사이트였음

알짜(Cash Cow) 그룹은 거래 규모와 빈도는 작지만 수익성은 높은 집단으로 나타났습니다.
즉, 큰 기업 = 좋은 기업 이 아니라, 작아도 효율이 높은 기업이 존재한다는 점을 확인했습니다.

4. 기존 일반 등급 내부에도 숨은 우량고객이 존재

특히 기존 일반 등급이면서 알짜로 분류된 고객이 다수 존재했고,
이들은 기존 등급만으로는 충분히 주목받지 못했을 가능성이 있습니다.

EDA 기반 추가 인사이트

알짜 그룹 중 기존 일반 등급 고객을 별도로 살펴본 결과 다음 특성이 확인되었습니다.

업종 분포는 제조업, 부동산업, 도소매업, 건설업 중심
디지털 채널 이용률은 낮고, 오프라인 채널 이용 비중이 높음
지역 분포는 대구·경북 비중이 높음
지역별 자금 성격은 상이하며, 일부 지역은 시설자금 중심, 일부 지역은 운전자금 중심

이는 단순 세그먼트 분류를 넘어,
지역·업종별 맞춤형 금융지원 전략과 디지털 전환 유도 전략까지 연결할 수 있음을 보여줍니다.

실무적 시사점
Segment	전략 방향
Star	유지 및 우대 전략 중심

특히 일반-알짜 고객군은 기존 안정성 등급만으로는 놓칠 수 있는 숨은 고효율 고객군으로,
중소기업 지원 전략에서 우선 검토 대상이 될 수 있습니다.

결론

이 프로젝트는 기존 안정성 중심 등급체계를 부정하는 것이 아니라,
이를 활동성과 수익성 관점에서 보완하는 세분화 모델을 제안했습니다.

그 결과, 동일한 기존 등급 내부에서도 실제 고객 가치 차이가 존재하며,
특히 기존 등급에서는 잘 드러나지 않던 숨은 우량 중소기업과 고효율 고객군을 발굴할 수 있음을 확인했습니다.

즉, 본 프로젝트는 중소기업 고객을 단순히 안정성으로만 보는 것이 아니라,

"얼마나 활발히 움직이는가" 와
"얼마나 효율적으로 수익을 만드는가" 를
함께 보아야 한다는 점을 데이터로 보여준 사례입니다.
